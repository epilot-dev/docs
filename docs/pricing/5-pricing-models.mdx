Pricing models define how customers are charged. Set the `pricing_model` field on a `Price` entity to one of:

1. `per_unit`
2. `tiered_volume`
3. `tiered_cumulative`
4. `tiered_flatfee`

Models are either unit-based or flat-fee-based:

## Standard

Charges a fixed amount per unit. When `variable_price=true`, the total is calculated from user input (e.g. consumption).

Example:

```json
{
  "name": "Standard",
  "pricing_model": "per_unit",
  "variable_price": true,
  "unit_amount_decimal": "0.055",
  "unit_amount": 6,
  "unit_amount_currency": "EUR"
}
```

```js
Consumption = 2000 kWh
Amount to pay = 2000 kWh * €0.055/kWh = 110 €
```

## Tiered Volume

The customer lands in a single tier based on total quantity, and that tier's unit price applies to **all** consumed units.

Example:

```json
{
  "name": "Tiered Volume",
  "pricing_model": "tiered_volume",
  "variable_price": true,
  "unit_amount_currency": "EUR",
  "tiers": [
    {
      "unit_amount_decimal": "0.055",
      "unit_amount": 6,
      "up_to": 1000
    },
    {
      "unit_amount_decimal": "0.054",
      "unit_amount": 5,
      "up_to": 2000
    },
    {
      "unit_amount_decimal": "0.053",
      "unit_amount": 5,
      "up_to": 3000
    },
    {
      "unit_amount_decimal": "0.05",
      "unit_amount": 5
    }
  ]
}
```

  style={{ width: '500px', borderRadius: '5px' }}
  src="/img/price-tiers-1.png"
  alt="Price Tiers"
/>

```js
Consumption = 2000 kWh
Amount to pay = 2000 kWh * €0.054/kWh = 108 €
```

## Tiered Cumulative

Each tier's unit price applies only to units **within that tier's range**. Lower tiers keep their price as consumption crosses into higher tiers.

Example:

```json
{
  "name": "Tiered Cumulative",
  "pricing_model": "tiered_cumulative",
  "variable_price": true,
  "unit_amount_currency": "EUR",
  "tiers": [
    {
      "unit_amount_decimal": "0.055",
      "unit_amount": 6,
      "up_to": 1000
    },
    {
      "unit_amount_decimal": "0.054",
      "unit_amount": 5,
      "up_to": 2000
    },
    {
      "unit_amount_decimal": "0.053",
      "unit_amount": 5,
      "up_to": 3000
    },
    {
      "unit_amount_decimal": "0.05",
      "unit_amount": 5
    }
  ]
}
```

<img
  style={{ width: '500px', borderRadius: '5px' }}
  src="/img/price-tiers-1.png"
  alt="Price Tiers"
/>

```js
Consumption = 2000 kWh
Amount to pay = 1000 kWh * €0.055/kWh + 1000 kWh * €0.054/kWh = 109 €
```

## Tiered Flat Fee

Charges a flat fee based on which tier the consumption falls into. The customer pays the tier's flat fee regardless of exact quantity.

Example:

```json
{
  "name": "Tiered Flat Fee",
  "pricing_model": "tiered_flatfee",
  "variable_price": true,
  "unit_amount_currency": "EUR",
  "tiers": [
    {
      "flat_fee_amount_decimal": "50.00",
      "flat_fee_amount": 5000,
      "up_to": 5
    },
    {
      "flat_fee_amount_decimal": "100.00",
      "flat_fee_amount": 10000,
      "up_to": 7
    },
    {
      "flat_fee_amount_decimal": "150.00",
      "flat_fee_amount": 15000,
      "up_to": 3000
    },
    {
      "flat_fee_amount_decimal": "200.00",
      "flat_fee_amount": 20000
    }
  ]
}
```

<img
  style={{ width: '500px', borderRadius: '5px' }}
  src="/img/price-tiers-2.png"
  alt="Price Tiers"
/>

```js
Max peak power = 7 kW
Amount to pay = 100 €
```

## Tier selection and input precedence

Tier ranges are defined by the `up_to` field. The customer's consumption determines which tier applies. For cumulative models, all tiers up to the matched tier contribute.

Consumption comes from two possible sources:

| Priority | Source | Where it's set |
| --- | --- | --- |
| 1 | Mapping input | NumberInputBlock in a [Journey](/docs/journeys/journey-builder), or mapping input on an [Order](/docs/pricing/orders) |
| 2 | Quantity input | ProductSelectBlock/ShoppingCart in a [Journey](/docs/journeys/journey-builder), or quantity input on an [Order](/docs/pricing/orders) |

If both inputs are provided, the **mapping input takes precedence**. If neither is provided, consumption defaults to `1`.
